# Thinking Like a Scientist? A Structural Study of LLM-Generated Research Methods

Replication package for the paper by Francesca Carlon, Brecht Verbeken, Vincent Ginis, and Andres Algaba.

This repository studies whether frontier LLMs systematically bias research-method suggestions. It compares methodological choices extracted from 1,000 recent LLM-related arXiv papers against suggestions generated by GPT-5.1, Gemini, and DeepSeek, then summarizes differences at the entity, taxonomy, and co-occurrence levels.

## Pipeline Scripts

The main scripts in `src/pipeline/` are:

- `01_batch_gt_extraction.py`: builds the ground-truth extraction batch from arXiv papers
- `02_batch_llm_suggestions.py`: creates suggestion batches from extracted research questions
- `03_normalize_entities.py`: normalizes extracted entity names and produces aggregate CSV summaries
- `04_compare_gt_llms.py`: compares model distributions against ground truth
- `05_batch_classification.py`: classifies extracted entities into taxonomy dimensions

Additional helper scripts include:

- `batch_claude_extraction.py`
- `batch_claude_classification.py`
- `compare_model_swap.py`
- `create_validation_samples.py`
- `create_blinded_annotation_package.py`
- `deterministic_provider_mapping.py`
- `fetch_arxiv_dates.py`
- `reconstruct_full_text.py`

Model-specific code lives in:

- `src/pipeline/gpt51_async/`
- `src/pipeline/gemini/`
- `src/pipeline/deepseek_async/`

- `openai`

```bash

.venv\Scripts\activate
pip install openai python-dotenv pandas numpy matplotlib seaborn thefuzz requests pymupdf arxiv scipy openpyxl anthropic
```

## Notes On Re-Running Scripts

This repository is currently artifact-first: the committed outputs are present, but some helper scripts still contain legacy path assumptions and may require small path fixes before a full end-to-end rerun from a fresh checkout.

In particular:

- scripts are not packaged as an installable project
- some scripts assume specific working-directory-relative paths
- not every intermediate input referenced by helper scripts is committed in the same location

The checked-in figures and tables should therefore be treated as the canonical outputs of this checkout.
