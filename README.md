# Thinking Like a Scientist? A Structural Study of LLM-Generated Research Methods

Replication package for the paper by Francesca Carlon, Brecht Verbeken, Vincent Ginis, and Andres Algaba.

This repository studies whether frontier LLMs systematically bias research-method suggestions. It compares methodological choices extracted from 1,000 recent LLM-related arXiv papers against suggestions generated by GPT-5.1, Gemini, and DeepSeek, then summarizes differences at the entity, taxonomy, and co-occurrence levels.

## Released Artifacts

The checkout already includes the main release artifacts:

- `figures/`: manuscript and appendix figure PNGs
- `tables/`: table exports in CSV and LaTeX format
- `data/analysis_outputs_names/`: normalized entity counts, percentages, and ratio summaries
- `data/batches/gt_extraction_input/`: saved OpenAI batch input for the 1,000-paper extraction run
- `data/batches/batch_suggestions_results/`: saved model outputs, classification outputs, comparison outputs, co-occurrence outputs, and Claude robustness outputs

No API keys are required to inspect the released figures, tables, aggregate CSVs, or saved batch outputs.

## Reproducibility Scope

This is an artifact-first replication package. The checked-in figures, tables, aggregate CSVs, and batch outputs are the canonical outputs for this release.

A full end-to-end rerun requires external model APIs and arXiv access. Results may not be bitwise reproducible because hosted model behavior, API defaults, model availability, and arXiv metadata can change over time. Some legacy scripts also contain working-directory-relative path assumptions, so use the committed outputs as the release reference unless you are intentionally rerunning or auditing a specific pipeline step.

Raw arXiv PDFs are not committed. The saved ground-truth batch input contains the title, abstract, and batch request metadata needed to inspect the corpus-level run; full-text reconstruction utilities are provided for robustness checks that require retrieved PDFs.

## Installation

Use Python 3.11 or newer and install into a virtual environment.

For artifact inspection and table or figure analysis:

```bash
python3 -m venv .venv
source .venv/bin/activate
uv pip install -e ".[figures]"
```

For API-backed extraction, suggestion generation, classification, and robustness workflows:

```bash
python3 -m venv .venv
source .venv/bin/activate
uv pip install -e ".[pipeline]"
cp .env.example .env
```

If `uv` is unavailable, replace the install command with:

```bash
python -m pip install -e ".[pipeline]"
```

## API Keys

Only scripts that call hosted model APIs require credentials. Create a local `.env` file from `.env.example` and fill in the keys needed for the workflow you run:

```env
OPENAI_API_KEY=your_key_here
GOOGLE_API_KEY=your_key_here
ANTHROPIC_API_KEY=your_key_here
```

The released artifacts can be inspected without creating `.env`.

## Pipeline Map

The main scripts in `src/pipeline/` are:

- `01_batch_gt_extraction.py`: builds the ground-truth extraction batch from arXiv papers
- `02_batch_llm_suggestions.py`: creates OpenAI suggestion batches from extracted research questions
- `03_normalize_entities.py`: normalizes extracted entity names and produces aggregate count and percentage summaries
- `04_compare_gt_llms.py`: compares model distributions against ground truth
- `05_batch_classification.py`: classifies extracted entities into taxonomy dimensions

Model-specific code lives in:

- `src/pipeline/gpt51_async/`: GPT-5.1 asynchronous suggestion workflow
- `src/pipeline/gemini/`: Gemini prompt and parsing utilities
- `src/pipeline/deepseek_async/`: DeepSeek prompt and parsing utilities

Validation and robustness helpers include:

- `reconstruct_full_text.py`: reconstructs sampled full-text records from arXiv PDFs
- `batch_claude_extraction.py`: reruns sampled ground-truth extraction with Claude Opus 4.6
- `batch_claude_classification.py`: reruns sampled taxonomy classification with Claude
- `compare_model_swap.py`: compares GPT-5.1 and Claude robustness outputs
- `deterministic_provider_mapping.py`: validates provider classifications with deterministic regex rules
- `create_validation_samples.py`: creates annotation samples for extraction, classification, normalization, and introducedness checks
- `create_blinded_annotation_package.py`: builds a blinded annotation package from validation samples
- `fetch_arxiv_dates.py`: fetches arXiv submission dates for corpus papers

## Useful Commands

The commands below assume the virtual environment is active.

List the released result files:

```bash
find figures tables data/analysis_outputs_names -maxdepth 2 -type f | sort
```

Run deterministic provider-mapping validation from the committed classification outputs:

```bash
python src/pipeline/deterministic_provider_mapping.py --output-dir outputs/provider_mapping
```

Run the Claude-vs-GPT model-swap comparison from the committed robustness outputs:

```bash
python src/pipeline/compare_model_swap.py --output-dir outputs/model_swap
```

Reconstruct full text for a sampled subset from arXiv:

```bash
python src/pipeline/reconstruct_full_text.py --n-papers 100 --seed 42 --sleep 3.0
```

Submit, poll, and retrieve a Claude extraction robustness batch:

```bash
python src/pipeline/batch_claude_extraction.py submit --n-papers 200 --seed 42
python src/pipeline/batch_claude_extraction.py poll --batch-id msgbatch_...
python src/pipeline/batch_claude_extraction.py retrieve --batch-id msgbatch_...
```

For the path-sensitive legacy GT-vs-LLM model-ratio script, run it from `data/batches/` so its relative paths resolve to `data/analysis_outputs_names/`:

```bash
cd data/batches
python ../../src/pipeline/04_compare_gt_llms.py
```
