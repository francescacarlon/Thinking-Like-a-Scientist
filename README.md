# Thinking Like a Scientist? A Structural Study of LLM-Generated Research Methods

Replication package for the paper by Francesca Carlon, Brecht Verbeken, Vincent Ginis, and Andres Algaba.

This repository studies whether frontier LLMs systematically bias research-method suggestions. It compares ground-truth methodological choices from 1,000 recent LLM-related arXiv papers against suggestions generated by GPT-5.1, Gemini 3 Pro, and DeepSeek-V3.2, then evaluates divergence at the entity, taxonomy, and co-occurrence levels.

## What is in this repo

The repository contains:

- the manuscript sources in `paper/`
- reproducible figure and table generators in `src/figures/`
- the API-backed data-collection and enrichment pipeline in `src/pipeline/`
- committed input data and intermediate artifacts in `data/`, `outputs/`, `figures/`, `tables/`, `paper/figures/`, and `paper/tables/`
- validation samples and blinded audit workbooks in `validation/`
- comparison artifacts in `figures_compare/`, `tables_compare/`, and `outputs_compare/`

## Reproduce the committed tables

The canonical entry point is [src/figures/generate_tables.py](C:/Users/Francesca%20Carlon/Desktop/Fran_stuff/Brussels/VUB/francesca_temp/src/figures/generate_tables.py).

```bash
python src/figures/generate_tables.py

```

`--check` reruns every registered table generator and fails if the regenerated files drift from the committed table artifacts.

## Artifact provenance audit

The repository includes a provenance registry for every figure and table referenced from the manuscript:

```bash
python src/figures/generate_artifact_provenance.py
python src/figures/generate_artifact_provenance.py --check
```

This produces:

- `outputs/artifact_provenance.csv`
- `outputs/artifact_provenance.md`

The check mode fails if manuscript references, registry rows, or committed provenance outputs are out of sync.

## Full pipeline reproduction

The full upstream collection and generation pipeline requires API credentials in `.env`.

Create `.env` from [.env.example](C:/Users/Francesca%20Carlon/Desktop/Fran_stuff/Brussels/VUB/francesca_temp/.env.example):

```env
OPENAI_API_KEY=your_key_here
GOOGLE_API_KEY=your_key_here
```

Core pipeline entry points live in `src/pipeline/`:

```bash
python src/pipeline/01_batch_gt_extraction.py
python src/pipeline/02_batch_llm_suggestions.py
python src/pipeline/03_normalize_entities.py
python src/pipeline/04_compare_gt_llms.py
python src/pipeline/05_batch_classification.py
```

Additional pipeline and audit helpers include:

- `src/pipeline/batch_claude_extraction.py`

- `src/pipeline/compare_model_swap.py`

- `src/pipeline/create_blinded_annotation_package.py`
- `src/pipeline/reconstruct_full_text.py`

Model-specific suggestion code is split across:
